### Перефразированный конспект лекций по веб-разработке

#### **HTML — Язык разметки текста**
**Определение HTML**  
HTML (HyperText Markup Language) — стандартизированный язык для формирования структуры веб-документов. Предназначен для описания содержимого страниц и указания браузерам способов визуализации информации.

**Создание HTML-документа**  
1. Сформируйте текстовый файл с расширением `.txt`.  
2. Переименуйте расширение на `.html`.  
3. Откройте файл в текстовом редакторе (например, Блокнот).  
4. Напишите HTML-код.

**Структура HTML-документа**  
HTML базируется на специальных элементах (тегах), заключённых в угловые скобки (`< >`). Теги делятся на парные (контейнеры) и одиночные.  
- Пример парного тега: `<B>Жирный текст</B>`.  
- Пример одиночного тега: `<p>` для нового абзаца.  

Основные разделы документа:  
- `HTML`: Обозначает тип документа.  
- `HEAD`: Включает метаданные и заголовки.  
- `BODY`: Содержит контент, отображаемый в браузере.  

Пример структуры:  
```html
<HTML>
  <HEAD>...</HEAD>
  <BODY>...</BODY>
</HTML>
```

**Содержимое раздела `HEAD`**  
- `TITLE`: Заголовок документа (отображается в браузере).  
- `LINK`: Устанавливает связи с внешними ресурсами.  
- `META`: Содержит метаданные (ключевые слова, описание).  
- `STYLE`: Определяет стили документа.  
- `SCRIPT`: Включает исполняемые скрипты.  

**Метаопределения**  
Используются для описания свойств документа.  
- Типы данных: `HTTP-EQUIV` и `NAME`.  
- Пример `HTTP-EQUIV`:  
  ```html 
  <META HTTP-EQUIV="refresh" content="10; URL=http://www.site.ru"> 
  ```  
  (перенаправление через 10 секунд).  
- Пример `NAME`:  
  ```html
  <META NAME="keywords" CONTENT="ключевое слово 1, ключевое слово 2">
  <META NAME="description" CONTENT="Описание документа">
  ```

**Раздел `BODY`**  
Ключевые параметры:  
- `BGCOLOR`: Цвет фона.  
- `BACKGROUND`: Фоновое изображение.  
- `TOPMARGIN`, `BOTTOMMARGIN`, `LEFTMARGIN`, `RIGHTMARGIN`: Отступы от границ.  
- `TEXT`: Цвет текста.  
- `ALINK`: Цвет активных ссылок.  
- `LINK`: Цвет непосещённых ссылок.  
- `VLINK`: Цвет посещённых ссылок.  

**Форматирование текста**  
- **Логическое форматирование**: Семантическое выделение элементов.  
- **Физическое форматирование**: Визуальное изменение текста.  

**Теги логического форматирования**  
- `<ACRONYM>`: Расшифровка аббревиатур.  
- `<CITE>`: Цитаты (курсив).  
- `<DEL>`: Удалённый текст (зачёркнутый).  
- `<EM>`: Акцент (курсив).  
- `<H1-H6>`: Заголовки.  
- `<STRONG>`: Важный текст (жирный).  

**Теги физического форматирования**  
- `<B>`: Жирный.  
- `<BIG>`: Крупный шрифт.  
- `<BLINK>`: Мигающий текст.  
- `<I>`: Курсив.  
- `<SMALL>`: Мелкий шрифт.  
- `<SUB>`: Нижний индекс.  
- `<SUP>`: Верхний индекс.  
- `<TT>`: Моноширинный.  
- `<U>`: Подчёркнутый.  
- `<FONT>`: Настройки шрифта:  
  - `FACE`: Шрифт.  
  - `COLOR`: Цвет.  
  - `SIZE`: Размер.  

**Структурное форматирование**  
Организация текста в логические блоки: абзацы, центрирование, отступы.  
- **Абзацы**:  
  ```html 
  <p ALIGN="left|center|right|justify">...</p>
  ```  
- **Текстовый блок**:  
  ```html
  <DIV>...</DIV>
  ```  
- **Отступ строки**: `&nbsp;` (неразрывный пробел).  
- **Перенос строки**: `<br>`.  
- **Запрет переноса**: `<NOBR>...</NOBR>`.  
- **Горизонтальный разделитель**:  
  ```html
  <HR WIDTH="..." SIZE="..." ALIGN="..." COLOR="..." NOSHADE>
  ```  

**Комментарии**  
```html 
<!-- Текст не отображается в браузере -->
```

**Гипертекстовые ссылки**  
- Базовая структура:  
  ```html 
  <A HREF="URL">Текст</A>
  ```  
- Примеры:  
  - Ссылка на сайт:  
    ```html
    <A HREF="http://www.tpu.ru">Томский политехнический университет</A>
    ```  
  - Email:  
    ```html
    <A HREF="mailto:mail@site.ru?Subject=Здравствуйте!">Отправить письмо</A>
    ```  
  - Файл:  
    ```html
    <A HREF="http://www.site.ru/docs/dogovor.doc">Скачать договор</A>
    ```  
  - Внутренние якоря:  
    ```html
    <A HREF="#font">FONT</A> и <A NAME="font"></A>
    ```  

**Списки**  
- **Нумерованные**:  
  ```html
  <OL TYPE="I|i|A|a" START="N">
    <LI>Элемент</LI>
  </OL>
  ```  
- **Маркированные**:  
  ```html
  <UL TYPE="disc|circle|square">
    <LI>Элемент</LI>
  </UL>
  ```  

**Таблицы**  
- Основные теги:  
  - `<TABLE>`: Контейнер таблицы.  
  - `<TR>`: Строка.  
  - `<TD>`: Ячейка.  
  - `<TH>`: Заголовок (жирный).  
  - `<CAPTION>`: Общий заголовок.  
- Параметры `<TABLE>`:  
  - `BORDER`: Граница.  
  - `CELLSPACING`: Пространство между ячейками.  
  - `CELLPADDING`: Отступ внутри ячейки.  
  - `ALIGN`: Выравнивание.  
  - `WIDTH`, `HEIGHT`: Размеры.  
- Параметры ячеек (`<TR>`, `<TD>`, `<TH>`):  
  - `ALIGN`: Горизонтальное выравнивание.  
  - `VALIGN`: Вертикальное выравнивание.  
  - `BGCOLOR`: Цвет.  
  - `BACKGROUND`: Фоновое изображение.  
  - `WIDTH`, `HEIGHT`: Размеры.  
  - `COLSPAN`, `ROWSPAN`: Объединение ячеек.  

**Добавление графики**  
```html
<IMG 
  SRC="путь" 
  BORDER="..." 
  WIDTH="..." 
  HEIGHT="..." 
  HSPACE="..." 
  VSPACE="..." 
  TITLE="..." 
  ALIGN="left|right|top|texttop|middle|absmiddle|bottom|baseline|absbottom"
>
```

#### **Лекция 1: Введение в Web**
**Темы курса**  
- Фундаментальные основы веба.  
- Продвинутый JavaScript.  
- Клиентская разработка (Vanilla JS).  
- Серверная разработка (Node.js).  
- Real-time сообщения (polling, WebSocket).  

**Базовый сценарий работы веб-приложения**  
1. Пользователь вводит URL.  
2. Браузер загружает HTML-документ.  
3. Браузер анализирует HTML и запрашивает дополнительные ресурсы.  
4. Браузер отображает страницу.  

**Структура URL**  
```
http://mi-ami.ru:8080/profile/account.html?gender=male&age=13#comments
```
- `http`: Протокол.  
- `mi-ami.ru`: Доменное имя.  
- `8080`: Порт.  
- `/profile/account.html`: Путь к документу.  
- `?gender=male&age=13`: Параметры запроса.  
- `#comments`: Якорь.  

**Документы**  
- **Типы (MIME)**: `text/html`, `text/css`, `image/png`, `video/mp4`.  
- **Классификация**:  
  - Статические: Постоянные файлы на сервере.  
  - Динамические: Генерируются при запросе.  

**HTTP**  
- Протокол для передачи данных поверх TCP.  
- Клиент-серверная модель.  

**Структура HTTP-запроса**  
- Метод (GET, POST и др.).  
- URL.  
- Заголовки.  
- Тело (опционально).  

**Структура HTTP-ответа**  
- Тело.  
- Заголовки.  
- Статус:  
  - 1xx: Информационные.  
  - 2xx: Успех.  
  - 3xx: Перенаправление.  
  - 4xx: Ошибка клиента.  
  - 5xx: Ошибка сервера.  

**HTML**  
- Спецификация: [htmlbook.ru/html](http://htmlbook.ru/html).  
- `DOCTYPE`: Определяет версию HTML. Для HTML5: `<!DOCTYPE html>`.

#### **Лекция 2: JavaScript**
**Основы**  
- Ресурс: [learn.javascript.ru/first-steps](https://learn.javascript.ru/first-steps).  

**Переменные**  
- `let`: Современное объявление.  
- `var`: Устаревшее.  
- `const`: Константа.  

**Типизация**  
- JavaScript — слабо типизированный язык.  

**Операции**  
- Математические: `+`, `-`, `*`, `/`, `%`, `+=`, `++`.  
- Сравнения: `>`, `>=`, `<`, `<=`, `==`, `===`, `!=`, `!==`.  

**Условия и циклы**  
- `if-else`, тернарный оператор `? :`.  
- `while`, `do-while`, `for`.  

**Функции**  
- Объявление:  
  ```javascript
  function имя(параметры) { ... }
  ```  
- Function Expression vs Declaration:  
  - Expression: Создаётся при выполнении.  
  - Declaration: Доступна до объявления.  
- Стрелочные функции:  
  ```javascript
  const sum = (a, b) => a + b;
  ```  

**Объекты**  
- Создание:  
  ```javascript
  let user = { name: "John", age: 30 };
  ```  
- Доступ: `user.name` или `user["age"]`.  
- Сравнение: По ссылке.  

**`this`**  
- Ссылается на объект, вызвавший метод.  

**Типы данных**  
- Примитивы: `string`, `number`, `boolean`, `symbol`, `null`, `undefined`, `bigint`.  
- Обёртки: `String`, `Number`, `Boolean` и др.  

**Числа**  
- Методы: `toString(base)`, `toFixed(digits)`.  
- Округление: `Math.floor()`, `Math.ceil()`, `Math.round()`, `Math.trunc()`.  
- Проверки: `isNaN()`, `isFinite()`.  
- Преобразование: `Number(value)`, `parseInt()`, `parseFloat()`.  

**Строки**  
- Интерполяция: `` `Привет, ${имя}!` ``.  
- Методы:  
  - `length`, `indexOf()`, `toLowerCase()`.  
  - `includes()`, `slice()`, `substring()`.  

**Массивы**  
- Методы:  
  - `pop()`, `push()`, `shift()`, `unshift()`.  
  - `slice()`, `concat()`, `forEach()`.  
  - `find()`, `filter()`, `map()`, `reduce()`.  
- Проверка: `Array.isArray()`.  

**Объекты: методы**  
- `Object.keys()`, `Object.values()`, `Object.entries()`.  

**Деструктуризация**  
- Массив:  
  ```javascript
  let [first, second] = arr;
  ```  
- Объект:  
  ```javascript
  let { title, ...rest } = options;
  ```  

**Коллекции**  
- `Map`: Ключ-значение.  
- `Set`: Уникальные значения.  

**JSON**  
- `JSON.stringify()`: Объект → строка.  
- `JSON.parse()`: Строка → объект.  

**Классы**  
- Синтаксис:  
  ```javascript
  class User {
    constructor(name) { this.name = name; }
    sayHi() { alert(this.name); }
  }
  ```  
- Наследование: `class Admin extends User`.  
- Приватные поля: `#privateField`.

#### **Лекция 3: Углубленный JavaScript**
**Продвинутые функции**  
- `Rest`: Сбор аргументов в массив.  
- `Spread`: Разворачивание массива в аргументы.  
- Замыкания: Функции, запоминающие внешние переменные.  

**Контекст**  
- `func.call(context, args)`: Вызов с контекстом.  
- `func.apply(context, [args])`: Вызов с массивом аргументов.  
- `func.bind(context)`: Привязка контекста.  

**Прототипы**  
- Наследование через `[[Prototype]]`.  
- `__proto__`: Устаревший доступ к прототипу.  
- `F.prototype`: Установка прототипа для `new F()`.  

**Модули (ESM)**  
- Экспорт: `export`, `export default`.  
- Импорт: `import`.  
- Динамический импорт: `await import()`.  

**Обработка ошибок**  
- `try/catch/finally`.  
- Объект ошибки: `name`, `message`, `stack`.  
- Кастомные ошибки:  
  ```javascript
  class ValidationError extends Error { ... }
  ```  

**Регулярные выражения**  
- Создание: `/шаблон/gmi` или `new RegExp()`.  
- Методы: `match()`, `test()`, `replace()`.  

**События**  
- Типы: `click`, `submit`, `keydown`.  
- Обработчики:  
  ```javascript
  element.addEventListener("click", handler);
  ```  
- Объект события: `type`, `clientX`, `preventDefault()`.

#### **Лекция 4: Архитектура Web**
**Этапы рендеринга**  
1. Загрузка HTML → DOM.  
2. Обработка CSS → CSSOM.  
3. Формирование Render Tree (DOM + CSSOM).  
4. Компоновка (Layout).  
5. Отрисовка (Paint).  

**Ограничения JS в браузере**  
- Нет доступа к ФС.  
- Ограниченные сетевые возможности.  
- Многопоточность через Web Workers.  

**NPM**  
- Менеджер пакетов для Node.js.  
- Инициализация: `npm init`.  
- Файлы: `package.json`, `package-lock.json`.  

**Архитектурные принципы**  
- DRY (Don’t Repeat Yourself).  
- KISS (Keep It Simple).  
- SOLID.  

**Декомпозиция**  
- Внутренняя связность (cohesion).  
- Слабая связанность модулей (coupling).  

**MVC**  
- **Модель**: Бизнес-логика, данные.  
- **Представление**: Визуализация данных.  
- **Контроллер**: Посредник между Моделью и Представлением.  

**Роутинг**  
- History API для SPA.  
- Паттерн "Медиатор".  

**CSS-архитектура**  
- Проблемы: Каскадирование, конфликты имён.  
- Решения:  
  - CSS-модули.  
  - CSS-in-JS (JSS).  
  - Методологии (SMACSS).

#### **Лекция 5: Node.js**
**Поиск сервера**  
- DNS → IP-адрес.  

**TCP-соединение**  
- Трёхэтапное рукопожатие:  
  1. `SYN` →  
  2. `SYN-ACK` →  
  3. `ACK`.  

**Клиент-серверные модели**  
- Статические страницы: Frontend + Backend.  
- SSR: Динамическая генерация на сервере.  
- Веб-сервисы: REST API.  

**Node.js**  
- Среда выполнения JS вне браузера.  
- Преимущества: Асинхронность, скорость, простота.  
- Недостатки: Отсутствие типизации, однопоточность.  

**NestJS**  
- Фреймворк для серверных приложений.  
- Поддержка TypeScript, DI, валидации.  

**TypeScript**  
- Статическая типизация, интерфейсы, декораторы.  
- Пример аннотации:  
  ```typescript
  let age: number = 30;
  ```

#### **Лекция 7: Асинхронный JavaScript**
**Event Loop**  
- Макротаски (setTimeout), микротаски (Promise), рендеринг.  
- Очередь выполнения: Call Stack → Event Loop.  

**Promise**  
- Состояния: `pending`, `fulfilled`, `rejected`.  
- Цепочки:  
  ```javascript
  fetch(url)
    .then(response => response.json())
    .catch(error => console.error(error));
  ```  
- Методы: `Promise.all()`, `Promise.race()`.  

**Async/Await**  
- Синтаксис:  
  ```javascript
  async function loadData() {
    const data = await fetch(url);
  }
  ```  

**Fetch API**  
- Пример:  
  ```javascript
  fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  });
  ```  

**Инструменты**  
- Транспиляция: Babel (ES6 → ES5).  
- Сборка: Vite, Webpack.
